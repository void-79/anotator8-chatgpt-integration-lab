# Security

## Current Status

This lab is a **local / demo read-only MCP integration**. It is **not** production-ready auth for student or customer data until OAuth 2.1 authorization-server implementation and deployment controls are added. When `MCP_AUTH_TOKEN` is unset, the server prints a 7-line ASCII banner to stderr on startup that says:

```text
================================================================
 WARNING: MCP_AUTH_TOKEN is unset.
 This mode is intended ONLY for local development (localhost).
 Any reachable network client can call all 8 read-only tools.
 BEFORE exposing on a public tunnel: set MCP_AUTH_TOKEN=<random>.
 See docs/SECURITY.md and docs/CHATGPT_APP_SETUP.md.
================================================================
```

This is intentional demo behavior. It is not a vulnerability in the strict sense (read-only tools, no FS writes, no shell, no secrets read), but it is a footgun for someone who runs the server on a public tunnel without setting the token.

## Data Handling

| Question | Answer |
| --- | --- |
| What data leaves the local machine? | Only data supplied to the MCP tools or typed into ChatGPT. The server itself does not upload files. |
| What data ChatGPT sees | Tool arguments, tool `structuredContent`, and text `content`. Widget-only `_meta` is intended for the component, not the model. |
| Are video files uploaded? | No. Video bytes are not read, uploaded, decoded, or previewed. |
| Is project JSON uploaded? | If the user passes project JSON to ChatGPT/MCP, ChatGPT and the MCP server process that JSON. Built-in fixture loading reads only `fixtures/sample-project.anotator8.json`. |
| Are local paths read? | No arbitrary path reads. Only allowlisted fixtures and widget source files are read. |
| Are secrets read? | No `.env`, SSH keys, cookies, browser profiles, or arbitrary filesystem paths are read. |

## Tool Safety

| Tool class | Status |
| --- | --- |
| Read-only tools | Implemented and annotated `readOnlyHint: true, destructiveHint: false, openWorldHint: false` |
| Write tools | Not implemented (deferred; must return reversible patches and require explicit user approval) |
| Mutation proposal tools | Disabled in `config/capabilities.example.json` |
| Arbitrary command execution | **Not present** — `grep` confirms zero `child_process` / `exec` / `spawn` calls in `src/server/**` |
| Broad filesystem access | **Not present** — `grep` confirms only `storage.ts` (fixture allowlist) and `widget-resource.ts` (widget source) call `readFile` |

## Controls

- **10 MB** maximum project payload enforced in `Anotator8Adapter.parse` with `IntegrationError("too_large_input", ...)`.
- **Optional Bearer auth** through `MCP_AUTH_TOKEN` (comma-separated tokens supported). When unset, a 7-line DEMO-ONLY banner is printed. When set, `auth.ts` enforces RFC 6750 `WWW-Authenticate: Bearer realm="anotator8-chatgpt-lab", error="..."` with 401 missing and 403 invalid.
- **OAuth 2.0 Protected Resource Metadata** (RFC 9728) at `/.well-known/oauth-protected-resource[/<path>]` (added in v0.3.0). The 401/403 challenge now also carries `resource_metadata="<well-known url>"` per RFC 9728 §5.1. Authorization-server implementation is a follow-up; this is the discovery foundation.
- **Production auth** must use OAuth 2.1 with protected resource metadata and per-tool scope checks.
- **Origin allowlist** defaults to `https://chatgpt.com` and `https://chat.openai.com`; local no-origin requests are allowed for smoke/Inspector. Add via `CORS_ORIGIN` (comma-separated).
- **Widget CSP** declares no external `connectDomains` or `resourceDomains`.
- **Audit logs** redact `Bearer <token>` and `MCP_AUTH_TOKEN=...` and truncate summaries to 500 chars (`audit.ts`).
- **Error responses** route through `IntegrationError` typed codes; raw error messages / absolute paths are not leaked in 500 responses (`app.ts`).
- **Tool output schemas** are Zod-typed and tested in `tests/contract/mcp-tool-contracts.test.ts`.

## Prompt Injection

Project text, labels, subtitles, and unknown fields are untrusted. Mitigations:

- Tools summarize and validate data deterministically.
- `suggest_labels` does not invent semantic labels.
- Reports include limitations that video bytes are not inspected.
- Future write tools must require explicit approval and return reversible patches.
- Server validates tool inputs even if generated by the model.

## MCP SDK Upstream Issue

`@modelcontextprotocol/sdk@1.29.0` + `@modelcontextprotocol/ext-apps@1.7.4` has a known recursion bug on `transport.onclose → server.close` for some error paths. The lab installs `process.on("unhandledRejection", ...)` in `app.ts` to capture the recursion into the audit log instead of letting it pollute test output. Tests still pass; the bug is non-fatal. Fix the root cause by upgrading the SDK once upstream resolves it.

## Remaining Concerns

| Risk | Mitigation | Remaining concern |
| --- | --- | --- |
| In-process OAuth 2.1 AS (v0.7.0) is suitable for self-hosted demos only | PKCE S256 enforced; DCR + CIMD supported; JWT validation checks sig + iss + aud + exp + nbf + scope. | Production must cut over to a real IdP (Auth0 / Okta / Cognito / Stytch). See `docs/OAUTH_AS.md` for the cutover recipe. |
| Demo bearer auth is weaker than OAuth | `MCP_AUTH_TOKEN` env sets a static-token allowlist; `MCP_OAUTH_REQUIRE_AUTH=true` forces JWT auth. The startup banner is unmissable in demo mode. | Lab must not be exposed to the public internet without `MCP_OAUTH_REQUIRE_AUTH=true` AND a production IdP. |
| Project JSON can contain sensitive education records | Read-only, no persistence, docs warn what ChatGPT sees. `isEducationRecord`, `dataResidency`, `ownerId`, `classroomId` are preserved as opaque fields. | User must decide whether to share project JSON with ChatGPT. |
| Widget receives hidden `_meta.projectData` for focus buttons | CSP locked down; no external network domains. `textContent` only (no `innerHTML`). | Remove or reduce `_meta.projectData` before production if not strictly needed. |
| Dependency vulnerabilities from `npm audit` | Noted; no forced major upgrade applied. | Needs dependency review before production. |
| MCP SDK recursion | Captured to audit log via `unhandledRejection` handler. | Wait for upstream fix. |
| Video bytes | Never read, never uploaded, never decoded. | n/a |
| Local FS | Allowlisted fixture path + widget source path only. | n/a |
| Shell exec | Zero `child_process` / `exec` / `spawn`. | n/a |
| OAuth well-known endpoints are unauthenticated by design (RFC 8414 §3 + RFC 9728 §3.1) | Return only public metadata (resource identifier, AS endpoints, scopes, bearer methods). No tokens, no PII. CORS `*` is appropriate for public discovery. | If the AS list is sensitive, the deployment must use a private AS. |
| In-memory AS state is lost on process restart | Authorization codes, registered clients, and the JWT signing key are all in-memory. | A process restart forces all clients to re-authorize. Acceptable for a demo; production IdP is required for high availability. |
