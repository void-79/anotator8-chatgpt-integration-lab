# Security

## Current Status

This lab is a local/demo read-only MCP integration. It is not production-ready auth for student/customer data until OAuth 2.1 and deployment controls are added.

## Data Handling

| Question | Answer |
| --- | --- |
| What data leaves the local machine? | Only data supplied to the MCP tools or typed into ChatGPT. The server itself does not upload files. |
| What data ChatGPT sees | Tool arguments, tool `structuredContent`, and text `content`. Widget-only `_meta` is intended for the component, not the model. |
| Are video files uploaded? | No. Video bytes are not read, uploaded, decoded, or previewed. |
| Is project JSON uploaded? | If the user passes project JSON to ChatGPT/MCP, ChatGPT and the MCP server process that JSON. Built-in fixture loading reads only `fixtures/sample-project.anotator8.json`. |
| Are local paths read? | No arbitrary path reads. Only allowlisted fixtures and widget source files are read. |
| Are secrets read? | No `.env`, SSH keys, cookies, browser profiles, or arbitrary filesystem paths are read. |

## Tool Safety

| Tool class | Status |
| --- | --- |
| Read-only tools | Implemented and annotated `readOnlyHint: true` |
| Write tools | Not implemented |
| Mutation proposal tools | Deferred; must return reversible patches and require explicit user approval |
| Arbitrary command execution | Not present |
| Broad filesystem access | Not present |

## Controls

- 10MB maximum project payload enforced in `Anotator8Adapter.parse`.
- Optional bearer token through `MCP_AUTH_TOKEN` for demo protection.
- Production auth must use OAuth 2.1 with protected resource metadata and per-tool scope checks.
- Origin allowlist defaults to `https://chatgpt.com` and `https://chat.openai.com`; local no-origin requests are allowed for smoke/Inspector.
- Widget CSP declares no external `connectDomains` or `resourceDomains`.
- Audit logs redact bearer tokens and truncate summaries.
- Tool output schemas are tested.

## Prompt Injection

Project text, labels, subtitles, and unknown fields are untrusted. Mitigations:

- Tools summarize and validate data deterministically.
- `suggest_labels` does not invent semantic labels.
- Reports include limitations that video bytes are not inspected.
- Future write tools must require explicit approval and return reversible patches.
- Server validates tool inputs even if generated by the model.

## Remaining Concerns

| Risk | Mitigation | Remaining concern |
| --- | --- | --- |
| Demo bearer auth is weaker than OAuth | Use only locally; configure `MCP_AUTH_TOKEN` for public tunnel demos | Production must implement OAuth 2.1 |
| Project JSON can contain sensitive education records | Read-only, no persistence, docs warn what ChatGPT sees | User must decide whether to share project JSON with ChatGPT |
| Widget receives hidden `_meta.projectData` for focus buttons | CSP locked down; no external network domains | Remove or reduce `_meta.projectData` before production if not needed |
| Dependency vulnerabilities from npm audit | Noted in QA; no forced major upgrade applied | Needs dependency review before production |
